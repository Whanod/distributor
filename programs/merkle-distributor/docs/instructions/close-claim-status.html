Close Claim Status Instruction

The close_claim_status instruction allows for closing a claim status account when it's no longer needed, reclaiming the rent.

Overview

This instruction enables closing a claim status account when certain conditions are met. Closing the account will reclaim the rent (SOL) paid for the account's storage back to the claimant.

Parameters

This instruction doesn't take any parameters directly, as all necessary information is derived from the provided accounts.

Accounts Required

  1. distributor (read): The Merkle Distributor account associated with the claim.
  2. claim_status (write): The claim status account to close.
  3. claimant (signer, write): The claimant who owns the claim status and will receive the reclaimed rent.
  4. system_program (read): The Solana System program.

Validation Checks

The instruction performs the following validations:

  1. Ensures both the distributor's and claim status' closable flags are set to true.
  2. Verifies the claimant is the signer and matches the claim status's claimant.
  3. Confirms that the claim status belongs to the specified distributor.

Close Process

If all validations pass, the instruction:

  1. Closes the claim status account.
  2. Returns the rent (SOL) to the claimant account.

Test Phase Restriction

This instruction is only available during the test phase of the Merkle Distributor program. It's intended for use during development and testing, not in production.

Security Considerations

  • The closable flag must be set to true in both the distributor and claim status to allow closing.
  • Only the designated claimant can close their own claim status.
  • The account closing is permanent and cannot be reversed.

Notes

  • This instruction helps in cleaning up after a claim is fully processed.
  • Users can reclaim the rent they paid when creating the claim status.
  • The admin should ensure that the closable flag is set appropriately when creating the distributor.
  • This instruction should be used with caution as it permanently removes the claim status from the blockchain, making it impossible to track past claims.
