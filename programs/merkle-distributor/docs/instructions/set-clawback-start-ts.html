Set Clawback Start Timestamp Instruction

The set_clawback_start_ts instruction allows the admin to set or update the timestamp when the clawback period begins.

Overview

This instruction enables the admin of a Merkle Distributor to configure when the clawback period starts. The clawback start timestamp determines when the admin can begin reclaiming unclaimed tokens from the distributor.

Parameters

  1. clawback_start_ts: The Unix timestamp (in seconds) when the clawback period begins.

Accounts Required

  1. distributor (write): The Merkle Distributor account to update.
  2. admin (signer): The admin of the distributor.

Validation Checks

The instruction performs the following validations:

  1. Verifies the admin is the signer and matches the distributor's admin.

Security Considerations

  • Only the admin can set the clawback start timestamp.
  • Setting a clawback start timestamp too early may not give users enough time to claim their tokens.
  • The admin should carefully consider the implications of the clawback timing on user expectations.

Notes

  • This instruction is typically used to give users a specific time window to claim tokens before unclaimed tokens can be reclaimed by the admin.
  • The clawback start timestamp can be updated multiple times by the admin as needed.
  • Once the clawback period has begun (current time >= clawback_start_ts), the admin can use the clawback instruction to reclaim unclaimed tokens.
  • Setting this timestamp to a far future date effectively disables the clawback functionality until that time.
