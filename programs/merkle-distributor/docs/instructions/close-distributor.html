Close Distributor Instruction

The close_distributor instruction allows for closing a Merkle Distributor account when it's no longer needed, reclaiming the rent.

Overview

This instruction enables closing a Merkle Distributor account when certain conditions are met. Closing the account will reclaim the rent (SOL) paid for the account's storage back to the admin.

Parameters

This instruction doesn't take any parameters directly, as all necessary information is derived from the provided accounts.

Accounts Required

  1. distributor (write): The Merkle Distributor account to close.
  2. admin (signer, write): The admin of the distributor who has authority to close it and will receive the reclaimed rent.
  3. token_vault (read): The token vault associated with the distributor.
  4. system_program (read): The Solana System program.

Validation Checks

The instruction performs the following validations:

  1. Ensures the distributor's closable flag is set to true.
  2. Verifies the admin is the signer and matches the distributor's admin.
  3. Confirms the token vault has a zero balance to prevent loss of tokens.

Close Process

If all validations pass, the instruction:

  1. Closes the distributor account.
  2. Returns the rent (SOL) to the admin account.

Test Phase Restriction

This instruction is only available during the test phase of the Merkle Distributor program. It's intended for use during development and testing, not in production.

Security Considerations

  • The closable flag must be set to true when creating the distributor to allow closing.
  • The token vault must have a zero balance, ensuring all tokens have been claimed or clawed back.
  • Only the designated admin can close the distributor.

Notes

  • This instruction helps in cleaning up after a distribution is complete.
  • If there are any tokens left in the vault, the close operation will fail.
  • The admin should ensure all claims are processed or perform a clawback before closing.
  • This instruction should be used with caution as it permanently removes the distributor from the blockchain.
