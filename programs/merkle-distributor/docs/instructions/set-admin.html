Set Admin Instruction

The set_admin instruction allows the current admin to transfer admin authority to a new account.

Overview

This instruction enables the current admin of a Merkle Distributor to transfer their admin privileges to a new account. The admin has authority over various aspects of the distributor, including clawback operations and other administrative functions.

Parameters

This instruction doesn't take any parameters directly, as all necessary information is derived from the provided accounts.

Accounts Required

  1. distributor (write): The Merkle Distributor account to update.
  2. admin (signer): The current admin of the distributor.
  3. new_admin (read): The account that will become the new admin.

Validation Checks

The instruction performs the following validations:

  1. Verifies the current admin is the signer and matches the distributor's admin.

Security Considerations

  • Only the current admin can transfer admin authority.
  • Admin privileges are powerful and should be transferred with caution.
  • The admin has authority to clawback tokens, change the clawback receiver, and perform other administrative functions.

Notes

  • This instruction is useful for transferring control of a distributor to another entity.
  • The new admin will have full control over the distributor after the transfer.
  • There is no way to recover admin access if transferred to an incorrect or inaccessible account.
