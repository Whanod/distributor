Claim Locked Instruction

The claim_locked instruction allows a claimant to claim their vested tokens according to the vesting schedule defined in the Merkle Distributor.

Overview

This instruction enables users to claim tokens that were previously set as "locked" during the initial claim process. The amount available for claiming is calculated based on a linear vesting schedule between the distributor's start and end timestamps.

Parameters

This instruction doesn't take any parameters directly, as all necessary information is derived from the provided accounts.

Accounts Required

  1. distributor (read): The Merkle Distributor account.
  2. claim_status (write): The claim status account tracking the user's claim.
  3. from (write): The token vault containing the tokens to distribute.
  4. to (write): The token account to send the claimed tokens to.
  5. claimant (signer): The account claiming the tokens, must be the owner of the to account and match the claimant in the claim status.
  6. token_program (read): The SPL Token program.

Vesting Calculation

The amount available for withdrawal is calculated using a linear vesting formula:

available = locked_amount * (current_ts - start_ts) / (end_ts - start_ts)

Where:

  • locked_amount is the total amount locked for vesting
  • current_ts is the current UNIX timestamp
  • start_ts is the start of the vesting period
  • end_ts is the end of the vesting period

The calculation is capped so that:

  • If current_ts < start_ts, available = 0
  • If current_ts >= end_ts, available = locked_amount

The instruction subtracts the already withdrawn amount from the calculated available amount to determine how much to transfer.

Validation Checks

The instruction performs the following validations:

  1. Ensures the distributor has not been clawed back.
  2. Verifies that the claimant in the claim status matches the signer.
  3. Confirms that the current timestamp is at least equal to the start timestamp.
  4. Checks that there are still locked tokens available to claim.

Token Transfer

If there are tokens available to claim, the instruction:

  1. Calculates the amount to transfer (based on the vesting formula).
  2. Updates the locked_amount_withdrawn in the claim status.
  3. Updates the total_amount_claimed in the distributor.
  4. Transfers the tokens from the distributor's vault to the claimant's token account.

Notes

  • Claimants can call this instruction multiple times as more tokens vest over time.
  • The maximum amount that can be claimed equals the original locked_amount.
  • If the distributor has been clawed back, this instruction will fail.
  • Vesting follows a linear schedule between the start and end timestamps.
