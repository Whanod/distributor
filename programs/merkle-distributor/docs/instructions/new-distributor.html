New Distributor Instruction

The new_distributor instruction creates a new Merkle Distributor instance that can be used to distribute tokens to multiple recipients efficiently.

⚠️ IMPORTANT SECURITY NOTICE

This instruction is susceptible to frontrunning attacks that could result in loss of funds if not handled properly.

An attack could look like:

  • A legitimate user opens a new distributor.
  • Someone observes the call to this instruction.
  • They replace the clawback_receiver, admin, or time parameters with their own.

One situation that could happen is the attacker replaces the admin and clawback_receiver with their own and sets the clawback_start_ts with the minimal time allowed. After clawback_start_ts has elapsed, the attacker can steal all funds from the distributor to their own clawback_receiver account.

How to avoid this attack:

  • When you call into this instruction, ensure your transaction succeeds.
  • After your transaction succeeds, read back the state of the created MerkleDistributor account and assert the parameters are what you expect, most importantly the clawback_receiver and admin.
  • If your transaction fails, double check the value on-chain matches what you expect.

Parameters

  • version: A version identifier for the distributor, used in the PDA derivation.
  • root: The Merkle root hash (32 bytes) that validates claims against the distributor.
  • max_total_claim: The maximum total amount of tokens that can be claimed from this distributor.
  • max_num_nodes: The maximum number of unique claims that can be made.
  • start_vesting_ts: The UNIX timestamp when vesting begins.
  • end_vesting_ts: The UNIX timestamp when vesting ends.
  • clawback_start_ts: The UNIX timestamp when the admin can clawback remaining tokens.
  • enable_slot: The slot number when claiming becomes enabled.
  • closable: Whether the distributor and claim status accounts can be closed.

Accounts Required

  1. distributor (write): The Merkle Distributor PDA account that will be initialized.
  2. base (signer): Base key used in the distributor's PDA derivation.
  3. clawback_receiver (write): Token account that will receive clawed back tokens.
  4. mint (read): The mint of the token to be distributed.
  5. token_vault (read): Associated token account owned by the distributor that holds the tokens.
  6. admin (signer, write): The admin wallet that pays for the transaction and has authority over the distributor.
  7. system_program (read): The Solana System program.
  8. associated_token_program (read): The SPL Associated Token program.
  9. token_program (read): The SPL Token program.

Validation Checks

The instruction performs the following validations:

  1. The start timestamp must be before the end timestamp.
  2. The start, end, and clawback timestamps must all be in the future.
  3. The clawback start timestamp must be after the end timestamp.
  4. The clawback start timestamp must be at least one day after the end timestamp.

PDA Derivation

The distributor account is a PDA derived with the following seeds:

  • "MerkleDistributor"
  • The base key
  • The mint key
  • The version (as a little-endian byte array)

After Initialization

After creating the distributor, you must deposit exactly max_total_claim tokens into the token_vault account. The program doesn't handle this transfer as part of initialization to avoid potential security issues with token program integration.
