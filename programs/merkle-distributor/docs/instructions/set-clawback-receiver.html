Set Clawback Receiver Instruction

The set_clawback_receiver instruction allows the admin to change the account that receives clawed back tokens.

Overview

This instruction enables the admin to update the clawback receiver token account for a Merkle Distributor. The clawback receiver is the account that will receive tokens during a clawback operation.

Parameters

This instruction doesn't take any parameters directly, as all necessary information is derived from the provided accounts.

Accounts Required

  1. distributor (write): The Merkle Distributor account to update.
  2. admin (signer): The admin of the distributor who has authority to change the clawback receiver.
  3. new_clawback_receiver (read): The token account that will receive clawed back tokens.

Validation Checks

The instruction performs the following validations:

  1. Verifies the admin is the signer and matches the distributor's admin.
  2. Ensures the new clawback receiver is initialized with the same mint as the distributor.

Security Considerations

  • Only the designated admin can change the clawback receiver.
  • The new clawback receiver must be initialized with the same mint as the distributor.

Notes

  • This instruction is useful if the admin needs to change where clawed back tokens will be sent.
  • The clawback receiver can be changed at any time before a clawback occurs.
  • After a clawback has occurred, changing the receiver has no effect.
